## 要解決的問題

作為一名全端工程師與財務分析師，我需要一個能夠展示多元技能與專案經驗的平台。市面上的作品集解決方案往往有以下限制：

- **缺乏技術深度**：多數平台無法展現完整的技術棧與架構思維
- **國際化支援不足**：無法同時支援繁中、英文、日文三種語言
- **內容管理不彈性**：難以整合 MDX 來撰寫技術文章與專案說明
- **效能不佳**：載入速度慢，影響使用者體驗與 SEO

我需要一個能夠完全掌控、具備現代化技術棧、支援多語言、且效能優異的作品集網站。

## 思考過程

### 技術選型

在選擇技術棧時，我考慮了以下面向：

**前端框架**：選擇 Next.js 15

- App Router 提供優秀的 Server Components 支援
- 內建 i18n 路由，簡化多語言實作
- 實驗性的 View Transitions API 支援，提升頁面轉場體驗

**內容管理**：選擇 fumadocs-mdx

- 完整的 TypeScript 支援與型別安全
- 靈活的 MDX 解析，支援自訂 React 元件
- 內建 meta.json 機制，方便管理內容順序

**UI 框架**：選擇 shadcn/ui + Tailwind CSS

- 元件可完全客製化，符合個人品牌風格
- 優秀的無障礙支援 (a11y)
- 性能優異的原子化 CSS

### 架構設計

採用 TDD（測試驅動開發）方法論：

1. **先寫測試**：撰寫 E2E、元件測試與單元測試
2. **紅燈階段**：確認測試失敗（功能尚未實作）
3. **綠燈階段**：實作功能，讓測試通過
4. **重構階段**：優化程式碼，保持測試通過

資料流設計：

```
MDX Files → fumadocs-mdx → projectsSource → Data Layer → Components
```

## 採用方案

### 核心功能

**1. 專案展示系統**

- 精選專案展示（Featured Projects）
- 專案詳細頁面
- 響應式設計（桌面 3 欄、行動單欄）
- View Transitions API 轉場動畫

**2. 多語言支援 (i18n)**

- 支援繁體中文 (zh-TW)、英文 (en)、日文 (ja)
- URL 路由包含語言代碼 (`/zh-TW/projects/[slug]`)
- 內容獨立管理，支援各語言獨立翻譯

**3. 效能優化**

- Static Site Generation (SSG)：所有頁面於 build time 生成
- Image Optimization：首張圖片使用 `priority` 載入（LCP 優化）
- 其餘圖片使用 lazy loading
- 使用 Next.js Image 元件自動優化圖片格式 (WebP/AVIF)

**4. 型別安全**

- 完整的 TypeScript strict mode
- Zod schema 驗證 frontmatter
- 自動產生型別從 fumadocs-mdx

### 技術棧

- **Framework**: Next.js 15.5.3
- **Language**: TypeScript 5.9
- **UI**: React 19.1.1, shadcn/ui, Tailwind CSS
- **MDX**: fumadocs-mdx 12.0.1, fumadocs-ui 15.8.2
- **Animation**: motion 12.23.22
- **Testing**: Jest, React Testing Library, Playwright
