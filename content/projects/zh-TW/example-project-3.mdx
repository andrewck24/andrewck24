## 要解決的問題

語言學習者在練習口說與寫作時，常遇到以下困難：

- **缺乏練習對象**：沒有母語人士可以對話練習
- **即時回饋不足**：無法立即知道文法或發音錯誤
- **學習動機低落**：缺乏有趣的互動與社群支持
- **教材不實用**：傳統教材與實際生活場景脫節

需要一個能夠連結學習者、提供即時回饋、且充滿互動樂趣的語言學習平台。

## 思考過程

### 使用者旅程分析

**目標用戶**：

- 想學習中文的日本上班族
- 想學習日文的台灣大學生
- 想學習英文的亞洲工作者

**核心需求**：

1. 找到語言交換夥伴（Language Exchange Partner）
2. 獲得即時的文法與發音回饋
3. 在真實情境中練習（不是背單字本）
4. 追蹤學習進度與成就

### 技術挑戰與解決方案

**挑戰 1：即時翻譯延遲**

- **問題**：傳統 API 呼叫翻譯服務延遲 > 2 秒
- **解決**：採用 WebSocket + 串流翻譯，延遲降至 < 500ms
- **技術**：Next.js Server-Sent Events (SSE) + OpenAI Streaming

**挑戰 2：多語言文法校正**

- **問題**：單一 LLM 對特定語言的文法校正準確度不足
- **解決**：針對中、英、日三語分別 fine-tune 專門模型
- **技術**：GPT-4 fine-tuning + 語言學專家標註資料集

**挑戰 3：發音評分**

- **問題**：語音辨識無法準確評估發音品質
- **解決**：結合 Speech-to-Text + Phonetic Analysis
- **技術**：Whisper API + 自建音素對比模型

## 採用方案

### 核心功能

**1. 語言交換配對系統**

- 智慧配對演算法（基於語言能力、興趣、時區）
- 即時聊天室（文字 + 語音）
- 內建翻譯與文法校正工具
- 對話主題建議（100+ 情境主題）

**2. AI 語言助教**

- 即時文法校正（自動標註錯誤並提供建議）
- 詞彙建議（更自然的表達方式）
- 發音評分（音素級別分析）
- 對話情境模擬（AI 扮演店員、面試官等角色）

**3. 社群學習功能**

- 每日學習挑戰（Streak 機制提升動機）
- 學習小組（Study Groups）
- 語言角落（Language Corner）：分享學習心得
- 排行榜與成就徽章系統

**4. 學習分析儀表板**

- 學習時數追蹤
- 詞彙量成長曲線
- 弱點分析（文法、發音、詞彙）
- 個人化學習建議

### 技術棧

**前端**

- Next.js 15, React 19, TypeScript
- Tailwind CSS, Framer Motion
- Web Speech API（語音輸入）
- WebRTC（即時語音通話）

**後端**

- Next.js API Routes（Server Actions）
- Prisma ORM + PostgreSQL
- Redis（即時訊息佇列）
- WebSocket (Socket.io)

**AI & ML**

- OpenAI GPT-4 API（翻譯、文法校正）
- Whisper API（語音辨識）
- Hugging Face Transformers（自訂模型）
- LangChain（AI workflow 編排）

**基礎設施**

- Vercel（前端部署）
- Railway（後端服務）
- Supabase（資料庫 + 即時訂閱）
- Cloudflare R2（媒體儲存）

### 技術亮點

**即時協作架構**

```
User A (Browser)
  ↕ WebSocket
Server (Next.js)
  ↕ Redis Pub/Sub
Server (Next.js)
  ↕ WebSocket
User B (Browser)
```

**AI Pipeline**

```
User Input → Whisper (STT) → GPT-4 (Grammar Check)
→ Translation → Response Generation → TTS (Text-to-Speech)
```
